Basic example

A simple text input which updates a single property when the onchange event fires. The property is displayed in two places but both are updated correctly. None of the code for handling this needs to concern itself with DOM manipulation, it's all handled by the framework.

HTML elements are marked with data-q-* attributes, where q is a handy namespace prefix that isn't too verbose. The framework looks for these attributes in the DOM and acts accordingly. The entire Javascript code for the above functionality looks like this:

import { directives } from '../src/reactive/index.js'

directives.load(document.getElementById('form'), {
	data: {
		textInput: 'write something',
	},

	clear() {
		this.data.textInput = ''
	},
})

The HTML looks like this:

<div>
	<input type="text" data-q-input="textInput">
	<button data-q-click="clear">Clear</button>
	<p>The first rendering of <code>textInput</code> (<strong data-q-text="textInput"></strong>)</p>
	<p>The second rendering of <code>textInput</code> (<em data-q-text="textInput"></em>)</p>
</div>

The data-q-input attribute looks for a textInput property within the data object and displays it as the input element's value property. When the onchange event fires it updates the value of the property with the current text of the input.

The data-q-text attribute displays the value of the textInput property of the data object using the innerText property of the element. It watches for changes to textInput and when that happens it sets the innerText property again.

The data-q-click attribute looks for a clear function on the root object and calls it when an onclick event is fired. In this case the clear function sets the textInput property to an empty string which updates the input element and the two display elements.
